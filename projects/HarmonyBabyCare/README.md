# 🍼 HarmonyBaby Care

> 基于 HarmonyOS NEXT 的母婴健康记录应用 UI/UX 设计方案

**产品定位**：隐私安全、智能高效的母婴健康记录工具

**核心差异**：深度整合鸿蒙原子化服务与小艺语音交互，实现"双手/双眼自由"的育儿记录体验

**一句话卖点**：*"不用打开App，不用放下宝宝，一句话完成宝宝成长记录"*

### 2. 喂养记录页

支持三种喂养方式：
- **母乳**: 双侧计时器，支持切换
- **奶瓶**: 快捷奶量选择 + 品牌 + 水温
- **辅食**: 食材标签 + 克重 + 过敏标记

### 3. 原子化服务卡片

| 卡片 | 尺寸 | 功能 |
|------|------|------|
| 喂养计时卡 | 2×2 | 动态计时器，支持开始/暂停/切换 |
| 今日概览卡 | 2×4 | 展示喂养/睡眠/尿布/体温统计 |
| 快速记录卡 | 1×4 | 一键快速记录四种类型 |
| 用药管家卡 | 2×2 | 倒计时提醒，标记已服用 |
| 生长曲线卡 | 2×2 | 身高体重趋势曲线 |

## 🎙️ 语音交互

### 支持的语音指令

```
"开始喂奶"        → 启动母乳计时器
"记一下喝了120ml奶" → 记录奶瓶喂养
"宝宝睡着了"      → 开始睡眠记录
"换尿布"          → 记录尿布更换
"今天吃了多少？"   → 查询今日喂养统计
```

### 语音 UI 状态

- **聆听中**: 波形动画 + 实时识别文字
- **处理中**: 加载动画
- **成功**: 绿色对勾 + 结果消息
- **失败**: 红色叉号 + 重试按钮

## 📐 编码规范

遵循 **HarmonyOS NEXT UI/UX Pro Max Skill** 编码规则：

### ✅ 必须遵守

1. **语言**: 使用 ArkTS，禁止 `any` 类型
2. **UI**: 使用 ArkUI 声明式语法
3. **状态管理**: 优先 `@State` → `@Prop` → `@Link` → `@Provide/@Consume`
4. **资源引用**: 禁止硬编码颜色和字符串

```typescript
// ✅ 正确
Text($r('app.string.feeding'))
  .fontColor($r('app.color.text_primary'))
  .backgroundColor($r('app.color.feeding'))

// ❌ 错误
Text('喂养')
  .fontColor('#2D3142')
  .backgroundColor('#FFB347')
```
